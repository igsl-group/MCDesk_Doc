# 4. Change Enablement

**Controlled Management of Changes**

## Major Features

| Feature | Description |
|--------|-------------|
| **Change Request Submission** | A structured way for users to propose changes to the IT infrastructure or services. |
| **Change Planning & Assessment** | Tools to define the scope, impact, risk, and resources needed for a change. |
| **Standard, Normal, and Emergency Change Approval Workflows** | Automated approval processes involving Change Advisory Boards (CABs) or designated approvers. |
| **Change scheduling and calendar to avoid conflicts** | Calendar or timeline views to manage the timing of changes and avoid conflicts. |
| **Change Implementation & Testing** | Features to track the execution of changes and record test results. |
| **Post-Implementation Review (PIR) tracking** | Functionality to capture lessons learned after a change is implemented. |
| **Customizable Change Advisory Board (CAB) approval processes** | Risk and impact assessment templates. |

## ITIL 4 Process / Practice Alignment

| Practice | Alignment |
|----------|-----------|
| **Change Enablement** | The entire feature set is designed to manage and control changes to IT services and infrastructure, minimizing disruption and risk. The core system for controlling the lifecycle of all changes, balancing the need for change with the need to protect services. |
| **Release Management** | Closely related to change enablement, as changes are often part of larger releases. |
| **Service Value Chain - Design & Transition, Deliver & Support** | Ensures changes are smoothly integrated into live environments. |
