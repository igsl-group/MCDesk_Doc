# 3. Problem Management

**Root Cause Analysis and Prevention of Recurrence**

## Major Features

| Feature | Description |
|--------|-------------|
| **Problem Identification** | Tools to identify potential problems based on recurring incidents, trends, or proactive monitoring. |
| **Root Cause Analysis (RCA)** | Features to facilitate investigations, including linking related incidents, analyzing logs, and documenting findings. |
| **Workaround Management** | Ability to document and communicate temporary solutions for incidents caused by known problems. |
| **Known Error Database (KEDB)** | A repository of known problems and their workarounds, accessible by service desk agents. |
| **Problem Closure** | Workflow for formally closing problems once a permanent fix or preventative action is identified. |
| **Works in conjunction** | Dedicated problem records linked to multiple incidents. |

## ITIL 4 Process / Practice Alignment

| Practice | Alignment |
|----------|-----------|
| **Problem Management** | Provides the tools to investigate the root causes of incidents and prevent recurrences. Directly supports this process by providing tools to investigate, diagnose, and resolve the underlying causes of incidents. |
| **Incident Management** | Problem management works in conjunction with incident management by leveraging incident data. |
| **Knowledge Management** | The KEDB is a vital part of the organizational knowledge base. |
| **Continual Improvement** | The entire practice is proactive and improvement-focused. |
